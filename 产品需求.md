## 〇、为什么做 / 为什么可以不做(诚实)

**人话:Swoonly = AI 生成的言情/狼人/霸总连载小说,一个价无限看——没有币、没有广告、不按章收割。** 直打整个品类被骂烂的"币太贵、看一本书要 $80–330"。

**为什么做(数据支撑):**
- **需求真 + 可变现**:≥10 个独立网文 app 同时在赚($4万–90万/月,簇合计 >$400万/月),不是个例。
- **头部守得烂 = 可打**:评分普遍 2.6–4.2(Lezhin 2.6 / Novelix 3.1 / MANGA Plus 3.62),评论中位才 1528、多 <2000 → 没人筑起口碑护城河。
- **窗口开着**:多个 NEW 新进就在赚(Novellair NEW $60万 / Lera NEW $10万 / Novelpack NEW $9万)→ 现在进场就能赚。
- **AI 内容 = 楔子**:对手靠授权/签约内容(贵、被迫收割);我们 AI 生成**零边际内容成本** → 薅得起也能**不薅**,把别人的成本中心变成我们的杀招。
- **差评指明方向**(实测 1-2★):骂**付费墙 83% + 广告 83%**,夸内容(骂"写得烂"仅 16%)——**用户不嫌内容,嫌被往死里薅。**

**为什么可以不做(诚实风险 + 化解):**
1. **AI 内容质量/同质化**(差评 #4 "灌水、AI 味"):化解——只做 AI 最擅长的类型文(言情/狼人/霸总);**预生成 + 人工过审 + 足长章节**,不灌水、不换皮重发。
2. **NSFW / 个人账号红线**(言情易擦边,头部多 17+):**化解(关键)**——定位 SFW、目标 **16+ 非 17+**;AI prompt 把尺度钉在"clean / fade-to-black";文案/截图零露骨词;不做 UGC。
3. **分发**:阅读私密、不自传播,品类龙头靠买量。化解——ASO 主打"无限读·无币·无广告"这条全网最大痛点 + 反贪婪口碑;承认要规模化可能得上轻量 ASA。
4. **Apple 4.3(同类 AI 故事 app 过多)**:化解——真差异化(无币无限模式 + 质量)+ 干净 listing,不做换皮。
5. **要内容后端**(不像纯本地 app):化解——**预生成的静态内容走 CDN**,零按用户推理成本,后端极轻。

## 七、内容引擎(AI,核心竞争力)

- **预生成内容库**(非按用户实时生成):LLM 批量生成连载 → **moderation 过滤(SFW)→ 人审抽检 → 存库 → CDN 静态分发**。
- **零按用户推理成本**:成本在一次性生成,摊给所有读者 ≈ 零边际 → 这就是"无限订阅还赚钱"的根。
- **体裁**:言情主线 + 狼人/命定之伴、霸总/亿万富翁、契约婚姻、黑道坏男孩、复仇重生、吸血鬼;次:幻想/修仙、LGBTQ+。
- **质量纪律**(打差评 #4/#8):足长章节、不灌水、不换皮重发、连贯人设;封面 AI 生成(BookTok 风)。
- **尺度纪律**(账号红线):prompt 钉死 clean/fade-to-black,过 16+ 不过 17+。

## 八、技术栈

- SwiftUI(iOS 17+);阅读器(分页/主题/字体)、SwiftData(书架/进度)。
- **轻内容后端**:预生成内容存对象存储 + CDN(静态 JSON/EPUB 分片),无按用户推理。⚠️ 这是唯一的后端,因内容分发不可避免;但极轻、近零 COGS。
- StoreKit 2 订阅;WidgetKit(追更)、App Intents(继续阅读)。
- Privacy Manifest;ITSAppUsesNonExemptEncryption=false。

## 八·五、前端设计(Quiet UI · Warm Paper / Soft Noir)

> 遵循 `工作流/最新UI设计风格.md`:**实色 surface,禁玻璃拟态/blur/material**;单主色;内容优先。

- **产品人格**:关键词 cozy/escape/binge;主隐喻=封面墙 + 夜读;视觉变体 **Soft Noir(深) + Warm Paper(浅)系统自适应**;与现有 app 不撞(rose 主色,书卷)。
- **核心 Token**:canvas `#F5F3EF`/`#101114`、surface `#FCFBF8`/`#181A1F`、elevated `#FFFFFF`/`#22252B`、text `#24221F`/`#F0EEE9`、**单主色 rose `#D45871`**(覆盖 <12%);发丝边 8%、卡片软阴影、圆角 16。阅读器三主题:Paper/Sepia/Night。
- **首页(Library)**:3 秒看懂"接着读哪本";唯一 CTA=继续阅读;Hero=continue-reading 行 + 进度条;其下 Saved。
- **关键流程**:首次成功=Browse→点封面→读前 3 章免费;锁章→Paywall;空状态有 CTA。
- **阅读器**:衬线(New York)正文、字号/主题可调、点击切沉浸、上下章;无广告无币。
- **动效/触感**:`.selection` 切章、`.success` 订阅成功;Reduce Motion 降级淡入。
- **可访问性**:Dynamic Type、VoiceOver(封面/锁标有 label)、Light/Dark 双对比、44pt。
- **平台**:系统 TabView/Form/sheet;封面=实色渐变块(非图、非玻璃);iOS 17+。

## 十、个人账号安全 + 红线
✅ **Books 类**,纯内容消费,无账户(仅进度)、不碰 Health/Finance/Gov、无 BLE/Web3。
⚠️ **NSFW 红线(成败关键)**:SFW 定位、**16+ 非 17+**、AI 内容钉 clean/fade-to-black、文案/截图零露骨、**不做 UGC**(无评论=免 1.2 moderation)、纯 Apple IAP。
⚠️ **4.3 防撞**:真差异化(无币无限)+ 质量 + 干净 listing,不做换皮矩阵。

## 十一·五、长尾扩张路线图(矩阵打法,分阶段)

> 目标:用"按体裁切分"吃掉网文赛道长尾流量。但**一次铺一堆换皮 app = Apple 4.3 spam,个人账号会被整号封**。所以分阶段、先验证、真差异化。

| 阶段 | 做什么 | 前提 |
|---|---|---|
| **P0 旗舰** | Swoonly 跑通(引擎+内容+能排+能赚) | 现在 |
| **P1 单 app 吃长尾** | keyword 字段铺满子体裁 + **Custom Product Pages**(一个 app 多落地页按体裁投) | P0 上线后 |
| **P2 真差异化分体裁 app** | 复用引擎,但**不同体裁库 + 不同品牌 + 不同视觉**(非换皮),**按序少量**上(狼人→霸总→幻想…) | P0 验证能赚 + 单个间隔开 |
| **P3 真矩阵放量** | 体裁 app 成组铺 | **迁公司账号**(个人账号别碰 4.3 红线) |

**铁律**:① 先验证再扩 ② 每个 app 真不一样(Apple 禁的是换皮群,不禁真不同的多 app)③ 放量前上公司账号。共用资产=AI 内容库(按体裁打标)+ 阅读器引擎。
